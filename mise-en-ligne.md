# Mise en ligne - L'Agriculture Fidélité

## Option rapide : version démonstration en ligne

Cette option publie la version actuelle telle quelle. Elle est utile pour montrer l'application, tester sur téléphone et faire valider le parcours.

Attention : les données restent stockées dans le navigateur. Ce n'est pas encore la vraie base clients commune.

Hébergeurs simples possibles :
- Netlify
- Vercel
- OVH / hébergement classique
- Hostinger
- Infomaniak

Fichiers à envoyer :
- index.html
- logo-agriculture.jpg
- manifest.json
- service-worker.js

## Option recommandée : vraie version clients

Pour un lancement officiel, il faut connecter l'application à une base de données.

À prévoir :
- base clients en ligne,
- compte administrateur sécurisé,
- ajout de points depuis plusieurs appareils,
- historique fiable,
- QR codes vraiment vérifiables,
- sauvegarde des données,
- page de confidentialité.

## Domaine conseillé

Exemples :
- fidelite.lagriculture-blois.fr
- carte.lagriculture-blois.fr
- app.lagriculture-blois.fr

## Ce dont j'ai besoin pour publier vraiment

1. Le nom de domaine que tu veux utiliser.
2. Le fournisseur du domaine ou de l'hébergement, par exemple OVH, IONOS, Hostinger, Wix, Squarespace, etc.
3. Savoir si tu veux d'abord une démo en ligne ou directement la vraie version avec base de données.

Je ne dois pas recevoir ton mot de passe en clair dans la conversation. Le mieux est que tu te connectes toi-même au compte, puis je te guide ou je te donne exactement quoi cliquer/configurer.
